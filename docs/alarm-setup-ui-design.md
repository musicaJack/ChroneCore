# 闹钟配置 UI  redesign（Apple 滚轮 + DSEG7 三区）

**状态：** 设计稿（待评审，**本阶段不改代码**）  
**硬件：** M5Stack Core2 横屏 320×240 · LVGL 9 · DSEG7 `chrone_font_dseg56`  
**关联文档：** [setup_alarm.md](./setup_alarm.md)（产品哲学）、[alarm-implementation.md](./alarm-implementation.md)（NVS/调度）、[clock-weather-layout.md](./clock-weather-layout.md)（顶栏/主区 Y）

---

## 1. 为什么要改

| 现状 | 问题 |
|------|------|
| 编辑页 `H+`/`H-`/`M+`/`M-` 小按钮 | 不像「设闹钟」，更像调参数；与表盘 **7 段数字钟** 视觉割裂 |
| 时间含义不直观 | 用户易误解为「再过几小时/分钟」而非「几点几分响」 |
| 无触觉反馈 | 与 [setup_alarm.md](./setup_alarm.md) 强调的 Apple 式「每步有反馈」不符 |

**目标：** 在设备上用 **与主时钟相同风格的 HH : MM : SS 三区**，以 **iPhone 闹钟滚轮** 的方式设置 **响铃时刻**，并配合 **震动（+ 可选 tick 音）**。

---

## 2. 产品原则（摘自 setup_alarm.md）

- **低认知负担**：半睡眠状态下也能操作；一眼看懂、一步完成。
- **时间是物理对象**：可拨动、有卡位、有阻尼/惯性，不是填表。
- **每步必有反馈**：视觉（高亮/滚动）+ 触觉（震动）+ 可选听觉（tick）。
- **克制、安静**：床头设备；动画与振动宜短、弱，不焦虑。

本方案在以上原则下，对齐 ChroneCore 已有 **DSEG7 主时间行** 与 **`vibration` 组件**（AXP192 LDO3）。

## 3. 时间语义（必读）

### 3.1 与调度层一致

当前 `chrone_alarm_t` 与 `chrone_alarm_check_tick()` 仅比较 **时、分**（整分钟到点），**不存秒**。

因此：

| 显示 | 用户理解 | 系统行为（推荐定稿） |
|------|----------|----------------------|
| **HH** | 24h 制「几点」 | 写入 `alarm.hour` |
| **MM** | 「几分」 | 写入 `alarm.minute` |
| **SS** | 与主钟一致的第三位 | **见 §3.2 二选一** |

### 3.2 SS 区策略（需产品确认）

**方案 A（推荐，P0）— 视觉一致、逻辑简单**

- 编辑页 SS **固定显示 `00`**，样式略暗（`#7A8499` 或 50% 透明度），**不可滚动**。
- 保存时忽略秒；到点在 `HH:MM:00`～`HH:MM:59` 内由现有 1 Hz 检查触发（与现网一致）。
- 优点：无需改 NVS；语义与 iPhone 闹钟一致（只设时分）。

**方案 B（可选，P1）— 三区皆可滚**

- SS 可设 0–59；扩展 NVS 或 `flags` 启用「秒级单次」；调度改为秒级或「下一 occurrence」计算。
- 优点：便于「1 分钟后」测试；缺点：实现与 Once/Daily 规则更复杂。

**文档后续默认采用方案 A**；若你坚持 SS 可调，在评审时改为 B。

### 3.3 不是「倒计时」

滚轮调整的是 **响铃时钟时刻**（例如 `07:30`），不是「30 分钟后」。可在编辑页副标题显示：

`Rings at 07:30` · `Next in 8h 12m`（可选，参考 setup_alarm.md 草案）

## 4. 视觉规范（对齐数字主钟）

### 4.1 字体与颜色

与 `digital_clock.cpp` 主时间行一致：

| 元素 | 字体 | 颜色 |
|------|------|------|
| 数字 | `chrone_font_dseg56` | 激活 `#FFB020`（`ui_color_seg_on`） |
| 冒号 | 同上 | 激活区 `#FFB020`；非激活列 `#4A5060` |
| 非激活列数字 | 同上 | `#7A8499`（`ui_color_muted`）或 40% 亮度 |

### 4.2 位置与尺寸

复用 `clock_layout.h`：

- 主滚轮区 Y：`CHRONE_TIME_Y`（与主钟同一垂直带，形成「还是在看时间」的连贯感）。
- 字高：`CHRONE_TIME_FONT_LINE_H`（57px）。
- 背景：全屏 `#06080C`，顶栏 36px、底栏可压缩或取消（编辑页可不用天气底栏）。

### 4.3 列布局（横屏 320px）

```text
┌──────────────────── 320 ────────────────────┐
│  Edit Alarm 1          [< List]             │  顶栏 36px
├─────────────────────────────────────────────┤
│              (CHRONE_TIME_Y)                │
│    ┌─────────┐   ┌─────────┐   ┌─────────┐   │
│    │   23    │ : │   28    │ : │   00    │   │  ← DSEG7 滚轮
│    │  (HH)   │   │  (MM)   │   │  (SS)   │   │
│    └─────────┘   └─────────┘   └─────────┘   │
│         ↑ 当前列高亮 + 略放大 1.05x（可选）    │
│  ───────────────────────────────────────────  │
│  Repeat:  Once                                │
│  Enabled  [====○]                             │
│              [ Save ]                         │
└─────────────────────────────────────────────┘
```

- **冒号** 不作为独立触摸区，仅装饰。
- **三区触摸热区**：每列宽约 **96px**（与两位数字 + 内边距匹配），高 **72px**（含上下「窥视」相邻数字的空间）。

## 5. 交互设计（参考 iPhone 闹钟）

### 5.1 选列

- **点按 HH / MM / SS 列** → 该列成为 **active**；另两列降亮。
- 同一时间仅一列可滚动（与 iOS 一致）。

### 5.2 拨动（上拨 / 下拨）

在 **active 列** 内：

| 手势 | 行为 |
|------|------|
| 向上滑动 | 数值 **增加** 1 步（HH +1h，MM +1min，SS +1s 若启用方案 B） |
| 向下滑动 | 数值 **减少** 1 步 |
| 快速滑动 | 带 **惯性**，多格减速停止（LVGL roller 或自研 `lv_anim`） |
| 松手 | **吸附**到最近整数格（detent） |

**循环：**

- HH：`0↔23` 循环（与 iOS 24h 一致）。
- MM / SS：`0↔59` 循环。

**不可选：** 用按钮 `H+`/`M-`；可保留 **点选数字直接键盘** 为 P2（Core2 无键盘可跳过）。

### 5.3 视觉反馈（滚动时）

- 当前值居中、最大、最亮。
- 上一档 / 下一档在上下露出约 **30%** 字号（iOS 滚轮「透视」），可用第二个 label 或 clip 容器实现。
- 滚动中列底可选 **细线指示**（caret）。

### 5.4 触觉反馈（Haptic）

使用现有 `components/vibration`：

| 事件 | 建议 |
|------|------|
| 每跨过 **1 个 detent**（时/分/秒变化 1） | `vibration_start(weak, 15~25ms)`，防连发 **≥50ms** 间隔 |
| 列切换（点 HH→MM） | 单次 `vibration_trigger()`（~100ms） |
| Save 成功 | 双脉冲（50ms + 80ms 间隔） |
| 无效操作（如 SS 锁定方案 A 仍拖动） | 无振动 |

封装建议：`chrone_haptic_detent()` / `chrone_haptic_confirm()`，避免 UI 层直接散落调用。

### 5.5 听觉（P1）

- 每个 detent 可选 **极短 tick**（`chrone_audio`，音量低于闹钟）。
- 夜间模式 Kconfig：仅振动、无声。

### 5.6 与主钟差异

| 主时钟 | 闹钟编辑 |
|--------|----------|
| 1 Hz 自动走时 | **冻结**显示值，仅手势改变 |
| 三区不可独立滚 | 三区独立滚轮（SS 按 §3.2） |
| 单击切换模拟/数字 | 本页不响应表盘切换 |

## 6. 页面与导航

### 6.1 进入配置

进入配置：**主区长按 ≥3s**（`tap_layer`，与单击切换表盘同层；长按抬手不触发切换）。  
（`alarm-implementation.md` 中的左+右 chord 不再作为主入口。）

### 6.2 列表页（简化）

```text
Alarms                    [< Back]
─────────────────────────────────
[ON]  07:30  Daily      >
[OFF] 08:00  Weekday     >
...
```

- 行内时间用 **DSEG14 或 Montserrat 22**（列表可略小）；进入编辑后用大 DSEG56。
- 点行 → 编辑页；Back → 时钟。

### 6.3 编辑页

| 控件 | 行为 |
|------|------|
| DSEG 三区滚轮 | §5 |
| Repeat | 行点击循环 Once / Daily / Weekday（可改为底部 segment） |
| Enabled | `lv_switch` |
| Save | 写 NVS + 触觉确认 + 回列表 |
| `< List` | 放弃未保存更改（可选二次确认，P2） |

**删除** 现有 `H+`/`H-`/`M+`/`M-` 按钮行。

### 6.4 默认初值

打开某槽位编辑时：

1. 若该槽已有配置 → 显示已存 `HH:MM`。
2. 若为新建/全 0 → 默认 **当前本地时间 + 1 分钟**（减少测试成本），秒 `00`。

## 7. 技术实现要点（供下一阶段编码）

### 7.1 组件划分

| 模块 | 职责 |
|------|------|
| `alarm_screen.cpp` | 列表 / 路由 / Save |
| `alarm_time_picker.cpp`（新建） | 三区滚轮 UI、手势、高亮列 |
| `chrone_haptic.c`（新建，薄封装） | detent / confirm 震动 |
| `chrone_alarm` | 不变（仍 `hour`/`minute`） |

### 7.2 LVGL 实现路线

| 方案 | 优点 | 缺点 |
|------|------|------|
| **A. `lv_roller` ×3** | 惯性/吸附内置 | 默认字体非 DSEG；需自定义 draw 或 overlay label |
| **B. 自研 drag + `lv_label`**（推荐） | 完全复用 `create_seg_part` / `style_seg_label` | 需实现惯性、snap、窥视两行 |
| **C. `lv_btnmatrix` 竖条** | 简单 | 不像 iOS |

**推荐 B**：与主钟同代码路径，长期可抽 `TimeColumnPicker` 复用到倒计时等。

### 7.3 触摸与底层热区

- 三区 **LVGL 透明容器** `clickable`，在 `tap_layer` 之上、`create_tap_layer` 之后 `move_foreground`（与日期双击相同策略）。
- 滚动手势在容器 `LV_EVENT_PRESSING` / `RELEASED` 计算 `dy`，换算步进；**不要**与 `chrone_input` 底栏冲突（编辑页可隐藏底栏逻辑或仅占底部 60px 外区域）。

### 7.4 调度与 NVS

- **不改** `alarm_cfg` blob 结构（方案 A）。
- `app_poll` 1 Hz 检查保持（配置页内也可响铃）。
- Save 后打印 `[i] ON HH:MM` 日志保留。
